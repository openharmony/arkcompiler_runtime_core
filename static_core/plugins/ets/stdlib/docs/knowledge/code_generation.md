# Code Generation Knowledge

This document only covers the constraints and workflow for template-based code generation. For build commands, see `build_and_test.md`.

## Core Model

```
Template file (.j2) → genlib.sh → Generated .ets file
```

- Template directory: `../templates/stdlib/`
- Generation script: `../templates/stdlib/genlib.sh`
- Generated files begin with `// NOTE: autogenerated file`

## Constraints

- **Do NOT** directly edit files that begin with `// NOTE: autogenerated file` — Reason: the next genlib run will overwrite your changes
- **You MUST** modify the corresponding template, then run `genlib.sh` to regenerate — Reason: the template is the single source of truth
- **You MUST** commit both the template changes and the generated files together — Reason: the build system verifies consistency; a mismatch causes build failure
- The build system validates consistency between templates and generated files; any inconsistency will cause the build to fail

## Regeneration Steps

```bash
export PROJECT=/path/to/static_core
chmod +x $PROJECT/plugins/ets/templates/stdlib/genlib.sh
$PROJECT/plugins/ets/templates/stdlib/genlib.sh
```

## Common Issues

| Issue | Cause | Solution |
| --- | --- | --- |
| Build failure | Template and generated file are out of sync | Run `genlib.sh` and commit both together |
| Edits lost | Directly edited an autogenerated file | Modify the template and regenerate |

## Pre-Modification Checklist

- Does the target file begin with `// NOTE: autogenerated file`? If yes, you **must** modify the template instead
- Have you run `genlib.sh`?
- Are the template and generated file committed together?
