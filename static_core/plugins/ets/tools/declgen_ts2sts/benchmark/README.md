# declgen_ts2sts — Benchmark Suite

End-to-end performance benchmarks for `declgen_ts2sts`. Measures wall-clock cost of
`Declgen.run()` across representative fixtures and aggregates the per-stage timings
exposed via `StageContext.stageTimings`.

## Layout

```
benchmark/
  README.md
  tsconfig.json            # extends ../tsconfig.json, adds benchmark/ to compile
  src/
    runner.ts              # CLI entry: `node build/benchmark/src/runner.js`
    Benchmark.ts           # warmup + iteration loop, stats (min/mean/median/p95/max/stddev)
    Reporter.ts            # console table + JSON file output
    registry.ts            # `defineBench(name, fn)` registry shared by scenarios
    types.ts
  scenarios/               # one `*.bench.ts` per workload
    declgen.bench.ts
  fixtures/                # input data, sorted by size
    small/
    medium/
    large/
  results/                 # JSON outputs (gitignored)
```

A scenario imports `defineBench` from `registry.ts` and registers one or more cases.
The runner iterates the registry, performs warmup runs, then measures `--iterations`
runs of each case, recording the end-to-end duration plus the per-stage breakdown.

Each fixture is registered twice:

- `declgen/<label> [full]` — cache disabled, every iteration runs the whole pipeline.
- `declgen/<label> [incremental]` — `setup()` wipes a per-case `cacheDir` and
  primes it once (cold run + emit). Every measured iteration then re-runs against
  the warm cache with no source changes — the best-case incremental scenario. The
  cache lives under `benchmark/results/_cache/<label>/`.

The runner prints a `Incremental vs full` summary table at the end pairing the
two variants by label.

## Run

```bash
npm run bench                              # full suite, default 10 iterations + 2 warmup
npm run bench -- --filter declgen          # only scenarios matching pattern
npm run bench -- --iterations 30 --warmup 5
npm run bench -- --json benchmark/results/run.json
```

### External real-world fixtures

A curated set of large open-source TypeScript projects (zod, rxjs, typeorm,
nestjs) is available as additional bench cases. They are **not** committed.
Fetch them once with:

```bash
npm run bench:fetch                        # shallow-clones pinned tags + npm install
FORCE=1 npm run bench:fetch                # re-clone, useful when bumping versions
```

The script lives at [benchmark/scripts/fetch-fixtures.sh](scripts/fetch-fixtures.sh).
Clones land in `benchmark/fixtures/external/<name>/` (gitignored). Scenarios
auto-detect each project; missing ones are skipped silently. Filter to a
single project with `--filter external/rxjs`.

## Conventions

- **Fixtures are checked in only when small.** Large fixtures should be generated by a
  script or pulled from `test/cookbook_tests/` to keep the repo light.
- **Each scenario is deterministic.** No clocks, RNG, or network inside the timed
  function; do setup outside the closure passed to `defineBench`.
- **Results go to `benchmark/results/`** which is gitignored. Commit summarised
  numbers in PR descriptions, not raw JSON.
- **Stage timings** come from `StageContext.stageTimings` (populated in
  [src/stages/Stage.ts](../src/stages/Stage.ts)). The reporter prints a per-stage
  mean alongside the end-to-end stats.
