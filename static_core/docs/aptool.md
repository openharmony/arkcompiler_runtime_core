# Ark Profile Tool (ark_aptool)

This document describes ark_aptool, a command-line utility for managing and analyzing `.ap` profile files generated by PGO (Profile-Guided Optimization).

## Overview

ark_aptool is a comprehensive tool for working with `.ap` profile files. These profile files contain runtime profiling data collected during application execution, including method execution patterns, inline caches, branch statistics, and throw data. The profiling information can be used to guide AOT compilation optimizations.

Currently, ark_aptool supports the following commands:

| Command | Description |
|---------|-------------|
| `dump` | Convert profile to human-readable YAML format |

## dump Command

The `dump` command converts a binary `.ap` profile file to human-readable YAML format.

### Usage

```
ark_aptool dump --ap-path <profile.ap> --output <output.yaml> [options]
```

### Options

| Option | Description |
|--------|-------------|
| `--ap-path` | Path to the input `.ap` profile file (required) |
| `--output` | Path for the output YAML file (required) |
| `--abc-path` | Path to an `.abc` file for metadata enrichment |
| `--abc-dir` | Path to a directory containing `.abc` files |
| `--keep-panda-file-name` | Filter: only dump matching panda files |
| `--keep-class-name` | Filter: only dump matching classes (requires `--abc-path`) |
| `--keep-method-name` | Filter: only dump matching methods (requires `--abc-path`) |
| `--help` | Print help message |

### Examples

Basic dump:
```
ark_aptool dump --ap-path profile.ap --output profile.yaml
```

Dump with ABC metadata (adds class descriptors, method signatures, and bytecode):
```
ark_aptool dump --ap-path profile.ap --output profile.yaml --abc-path app.abc
```

### Output Format

The dump command produces YAML output with the following structure:

```yaml
ClassCtxStr: "<class context string>"

PandaFiles:
  - index: 0
    file_name: "app.abc"

Methods:
  - panda_file_index: 0
    entries:
      - method_index: 123
        class_index: 45
        class_descriptor: "Lapp/MyClass;"
        method_signature: "foo:(f64)"
        inline_caches:
          - pc: 0x10
            bytecode: "call.virt.short v0, v1"
            targets:
              - class_index: 67
                panda_file_index: 0
                class_descriptor: "Lapp/Target;"
              - megamorphic: true
        branches:
          entries:
            - pc: 0x20
              bytecode: "jnez v0, +8"
              takenCount: 100
              notTakenCount: 5
        throws:
          entries:
            - pc: 0x30
              bytecode: "throw v0"
              takenCount: 2
```
